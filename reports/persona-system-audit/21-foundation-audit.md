## RANKED FOUNDATION GAPS THAT WOULD BLOCK INTEGRATION

1. **criome ↔ contract feature-branch pins (HIGH).** criome depends on `signal-criome`/`meta-signal-criome` at `criome-authorization-push`, not `main`. Producers must land on main before criome can integrate cleanly. Concrete: `criome/Cargo.toml`.
2. **schema-next trait/impl emission leg not integrated + nota-next `#[shape]` derive missing (HIGH, coupled).** Any schema-derived component needing method-bearing traits or generic impl-headers can't be fully generated yet. Blocker chain: nota-next `#[shape]` derive (`nota-next` §240) → schema-next `{| |}` reconciliation (`schema-next` §818-830) → schema-rust-next emission. Honestly flagged, but real.
3. **schema-rust-next `.expect` panics at the token-parse boundary (MEDIUM, typed-error discipline).** `src/lib.rs:2186/2955/5844` (and `parse2` at `:6062`) panic if schema data lowers to an unparseable type/import/expr string, instead of returning the crate's typed `Error`. Acceptable-loud for a generator, but a schema-driven input can crash it rather than surface a typed diagnostic.
4. **criome composite `Composition` evaluation unimplemented (MEDIUM).** `AllOf/AnyOf/Threshold/WorkflowStep` inside a `Composition` escalate to Psyche instead of evaluating (`src/language.rs`). Safe default, functional gap, under-documented.
5. **criome push side absent (MEDIUM, honestly flagged).** Subscription registry tracks tokens but emits no `IdentityUpdate`/authorization push events (`src/actors/subscription.rs:17-20`) — violates the workspace push-not-pull target until landed.

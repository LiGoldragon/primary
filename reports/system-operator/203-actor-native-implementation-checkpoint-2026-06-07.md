## Lojix Runtime Shape

The `lojix-daemon` now uses `ActorMultiListenerDaemon` with two authority-tiered listeners:

- ordinary listener for `signal-lojix` reads and subscriptions;
- owner/meta listener for `meta-signal-lojix` deploy and retention mutations.

Each listener has actor-runtime request admission through `RequestGate`. Accepted connections are Tokio Unix streams and frame IO uses `read_body_async` / `write_body_async`.

The generated `NexusEngine` trait is still synchronous. To keep synchronous runner and Nix process effects off the async accept path, each request execution is isolated with `tokio::task::spawn_blocking`. This is the correct local bridge until the generator/runtime grows an async effect trait.

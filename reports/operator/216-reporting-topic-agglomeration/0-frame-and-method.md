# 216 — reporting-topic-agglomeration frame

## Frame

Psyche asked for report topic agglomeration while clarifying the schema macro
stack. The implementation work continues locally in `nota-next` and
`schema-next`; the parallel report lane surveys recent operator/designer report
topics and proposes how topic-prefixed filenames should group future reports.

## Sub-agent assignment

- `1-report-topic-survey.md` — survey recent schema/NOTA/operator/designer
  reports, identify durable topic clusters, and recommend a short filename
  topic-prefix convention. Do not edit code or skills.

## Method

Use the current reporting discipline as the baseline. Treat report topic
agglomeration as forward-only unless a report is already being superseded or
refreshed. Do not rename old reports in bulk.

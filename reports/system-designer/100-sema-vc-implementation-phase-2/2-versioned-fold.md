## New surface

- **`Engine::checkpoint()`** (`src/engine.rs:1219`): folds latest-checkpoint rows + log suffix, chunks the sorted view into 1 MiB content-addressed `CheckpointSegment`s, writes `CheckpointMetadata` + segments + latest-checkpoint counter in one transaction. Returns `CheckpointReceipt`. Refuses incomplete logs (`VersionedLogIncomplete`, counted commit-log vs versioned-log) and empty coverage (`CheckpointNothingToCover`). **Logs no versioned entry** (documented decision, see designDecisions). `Engine::latest_checkpoint()` (`src/engine.rs:1311`) loads and `verify()`s the portable artifact.
- **`ImportSession`** (`src/import.rs`): minted only by `Engine::begin_import()` (`src/engine.rs:1395`) on a fresh store; holds `&mut Engine` so ordinary mutation handlers structurally cannot interleave. `ingest_checkpoint` verifies all content addresses; `commit(&dyn FamilyDirectory)` applies everything in one transaction (`Engine::apply_import`, `src/engine.rs:1406`): catalog + identified counters restored verbatim from the inventory, checkpoint rows installed so later checkpoints chain, suffix entries inserted verbatim through the same `insert_versioned_row` choke point as live writes (preserving sequences, digests, chain, tombstones; outbox rows included), metadata commit-log entries derived via `impl From<&VersionedCommitLogEntry> for CommitLogEntry` (`src/log.rs:62-86`), cursors set to the suffix end, and the folded view materialized via `RowMaterializer` — never through assert/mutate. Internal-consistency guard: the metadata's store schema hash must re-derive from its own carried inventory (`impl From<&[FamilyIdentity]> for StoreSchemaHash`, `src/versioning.rs:139-148`).
- **`Engine::rebuild_from_log()`** (`src/engine.rs:1349`): fold (checkpoint rows +) log, then one transaction applying tombstones for touched-not-kept keys followed by final rows. Touched-key clearing is a full clear by construction because no write path bypasses the log. Returns `RebuildReceipt` (`src/fold.rs` — view digest, row/cleared/folded counts). `replay_versioned` remains the per-family surface; its tombstone-under-Assert/Mutate error is now the typed `ReplayTombstonePayload` (review fix b, `src/engine.rs:1199-1203`).
- **Mirror outbox** (`src/outbox.rs` + `src/engine.rs:1488-1600`): `OutboxEntry` (sequence + entry digest) written by `insert_versioned_row` (`src/engine.rs:1947-1958`) — the single choke point all seven write paths and import route through. API: `unshipped_outbox()`, `mirror_head()`, `acknowledge_mirror(MirrorHead) -> MirrorAcknowledgement::{Advanced,Unchanged}` (idempotent; `MirrorHeadForked` / `MirrorHeadUnknown` / `OutboxEntryMismatch` typed errors), `durability_of(CommitSequence)` and `store_durability()` returning `Durability::{LocalCommitted, QueuedForMirror, ServerCommitted}`. Library-only; the mirror actor loads payloads via `versioned_replay_from_sequence`.
- **Review fix (a)**: `Engine::open` (`src/engine.rs:55-86`) validates layout slot, engine-history heuristic, AND catalog decode before any write; the layout stamp lands only after validation on a virgin store. Witness: `rejected_open_does_not_stamp_the_store_it_rejects` (`tests/family_identity.rs`).
- **Storage layout 3** (`src/engine.rs:36-45`): layout-2 stores carry versioned entries with no outbox rows — a mirror would silently ship incomplete history (29pb: state loss unacceptable), so they hard-fail typed. `LAYOUT_BEFORE_SLOT = 1` keeps slotless-store reporting truthful.

## designDecisions
KEPT from the recovered work (judged sound): the digest-verifies/segment-restores split and every checkpoint type (CheckpointMetadata field set incl. covered_entry_digest as the suffix chain head, FamilyInventory with IdentifiedCounter rows, content-addressed CheckpointSegment); the CanonicalView fold with semantic ViewKey (table coordinate excluded, matching StoreSchemaHash discipline) and the touched-set tombstone-clearing design; RowMaterializer as a one-shot, engine-minted, transaction-scoped affordance (no general write authority escapes); the FamilyDirectory trait as the component's typed family knowledge; the error Carrier smuggling typed errors through sema::Error::Io across the kernel's closure boundary; the rich recovered Error vocabulary (used every variant except adding CheckpointRowMissing). REVISED: removed dead recovered methods FamilyInventory::contains_family and CanonicalView::row_count (their jobs are done by into_rows resolution and materialize counts); realized the recovered pub(crate) opening of EntryDigest::from_entry_fields as fold-time per-entry digest recomputation, which also made a separate suffix store-name/schema check unnecessary (the digest cryptographically binds those fields — and suffix entries legitimately carry differing store schema hashes when families registered mid-history, so an equality check would have been wrong); did NOT use CheckpointSchemaMismatch against suffix entries for that reason, only as the metadata-vs-own-inventory internal-consistency guard. CHECKPOINT-NOT-LOGGED: checkpoints are derived artifacts of the log — the log already contains everything the checkpoint folds; logging the fold would make history describe itself, perturb the digest chain with non-state entries, and force every replayer to skip them. Documented in ARCHITECTURE.md and witnessed (no sequence advance, no log growth). IMPORT-SESSION NOUN: ImportSession minted only by Engine::begin_import on a fresh store, holding &mut Engine — the exclusive borrow is the structural unavailability (no typestate needed; the borrow checker carries the invariant per skills/rust/methods.md). Ingestion is two typed steps (one verified checkpoint, ordered suffix) and one atomic commit. SEGMENT CHUNKING: kept the recovered deterministic walk at a 1 MiB soft budget, at least one row per segment — redb loads each value whole, so chunking bounds every segment read/write; determinism keeps segment digests stable for identical views. LAYOUT 2-vs-3: bumped to 3 and hard-fail. A layout-2 store under this code is incorrect, not additive: its versioned entries have no outbox rows, so the unshipped suffix would silently omit them and a mirror would ship an incomplete history — exactly the state loss 29pb forbids. Pre-production, no compatibility shim: typed StorageLayoutMismatch, rebuild through import/replay. Added LAYOUT_BEFORE_SLOT=1 so slotless legacy stores report stored:1 truthfully instead of the old STORAGE_LAYOUT-1 arithmetic. OUTBOX ROW SHAPE: lean (sequence + entry digest) — the versioned entry is the shipped payload, the row is only the queue position plus the digest the acknowledgement must echo; acknowledge cross-checks outbox row against the logged entry (OutboxEntryMismatch) before fork detection. Import writes outbox rows for the suffix (the layout invariant: every versioned entry has its row) but deliberately does not restore the shipped cursor — acknowledgement is a transport fact a restore cannot fabricate; re-acknowledgement is idempotent. VERSION: 0.4.0 (minor: new public surface + breaking storage change, pre-1.0 per skills/versioning.md; no major without psyche authorization).

Recommend follow-up witnesses that doctor a logged entry, a segment, and an acknowledgement digest.
- Identified-record keys ride RecordKey as decimal strings and get parsed back in RowMaterializer::apply_identified (src/fold.rs:417-426, typed MaterializeIdentifierParse error). This is string-typification of RecordIdentifier — the versioned-log key position wants a sum type (domain key | identifier). The wire shape predates this branch, but pre-production rules permit breaking it; worth a beads item.
- CommitLogOperation::new was weakened from TableName to impl Into<String> (src/log.rs:92) to serve the new From<&VersionedLogOperation> projection — a public constructor lost its typed-domain guard. A pub(crate) string-accepting path or a From impl on the operation alone would have preserved it.
- store_durability() of a versioned store with an empty log reports ServerCommitted (src/engine.rs:1620-1629, 'an empty log is trivially mirrored') — vacuously true and documented, but a component reading durability before its first write may be misled; consider a distinct level or doc emphasis at the component layer.
- identified_counter_inventory recovers table names by stripping a string suffix from counter keys (src/engine.rs:1642-1653, shared IDENTIFIED_COUNTER_SUFFIX const). Stringly inventory recovery; a typed counter table would dissolve the parse.
- Checkpoint::verify's segment-count mismatch arm returns SegmentMissing with a zeroed placeholder digest when the artifact carries MORE segments than referenced (src/checkpoint.rs:504-513) — the error shape conflates missing/extra and can report an all-zero digest.
- CheckpointMetadata::new takes 10 positional arguments under #[allow(clippy::too_many_arguments)] (src/checkpoint.rs:357-385); pub(crate) and all distinctly typed, but a grouped sub-struct (coverage triple: range + snapshot + entry digest) would read better.
- acknowledge_mirror validates in a read transaction and writes the cursor in a separate write transaction (src/engine.rs:1561-1592) — safe only under the documented single-owner Engine constraint (ARCHITECTURE.md 'single-owner handle'); consistent with the crate's prevalidation idiom but a known sharp edge for the future mirror actor.

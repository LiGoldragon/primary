## The thesis up front: "the schema daemon" is a category mistake — and the real running daemon already exists

The brief asks "what would the schema daemon be — a running schema service?"
The honest answer from the source: **the schema stack is correctly a
BUILD-TIME library, not a runtime daemon, and it should stay that way.** The
psyche's phrase "the schema daemon" most accurately resolves to **the
schema-DERIVED daemon** — a component daemon whose Signal/Nexus/SEMA types are
generated by this stack at build time. That daemon is not hypothetical: it is
`spirit/` (the schema-derived Spirit pilot), and it BUILDS AND RUNS TODAY.

Concretely verified end-to-end this session:

- `spirit/` has three plane schema files (`schema/signal.schema`,
  `schema/nexus.schema`, `schema/sema.schema`, all dated Jun 5 13:37) and
  generated `src/schema/{signal,nexus,sema}.rs` emitted from them by
  `schema-rust-next`'s build driver (`spirit/build.rs` uses
  `GenerationPlan::new(...).with_module(signal_runtime_module).with_module(nexus_runtime).with_module(sema_runtime)`).
- `cargo build --bins` succeeds: it pulls `nota-next`, `schema-next`
  (`#5311f9ad`), `schema-rust-next` (`#2437413f` — the COMMITTED head),
  `triad-runtime`, regenerates schema, and compiles `spirit` +
  `spirit-daemon`. Finished in ~5s, clean.
- `cargo test --features nota-text --test process_boundary`: 6/6 pass,
  including `cli_and_daemon_exchange_nota_over_rkyv_socket`,
  `daemon_persists_sema_file_across_a_restart`, and
  `candidate_daemon_handover_from_production_copy_preserves_original_sema_database`.

So a single schema-derived component daemon, fed by this entire area, is a
live, tested, running process. The forward work is NOT "make the schema a
daemon" — it is "make schema-rust-next emit the daemon binary glue so each
component daemon shrinks to near-nothing, then bring the rest of the triad
components up under persona supervision."

   stable slice), but structure the emitter so the multi-listener model can be
   added without rework. Scaffold now, finalize the concurrency surface later.

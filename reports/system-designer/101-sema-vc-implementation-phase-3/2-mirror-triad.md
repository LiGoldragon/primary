## signal-mirror (the shipper/restorer vocabulary)

`schema/lib.schema` (signal-mirror:1) declares four domain-verb operations and seven past-tense replies plus a typed fault: `Append(EntrySuffix)` -> `Appended(AppendReceipt) | AppendRejected(AppendRejection{UnknownStore|SequenceGap|HeadForked|DigestMismatch|EmptySuffix, current head})`; `PublishCheckpoint(CheckpointArtifact)` -> `CheckpointPublished | PublishRejected{UnknownStore|CoverageRegressed}`; `Restore(RestoreQuery)` -> `Restored(RestoreBundle{checkpoint, suffix}) | RestoreRejected{UnknownStore|NoCheckpoint}`; `ObserveHeads(HeadQuery)` -> `HeadsObserved(HeadListing)`; `MirrorFaulted(FaultReport)`. `build.rs` is a three-line `ContractCrateBuild` (per schema-rust-next INTENT); generated module at `src/schema/lib.rs` (934 lines); `src/lib.rs` adds only small accessors on the generated nouns. `tests/round_trip.rs`: 14 witnesses, every operation through the length-prefixed rkyv frame AND the NOTA text surface, plus an opaque-bytes witness.

## meta-signal-mirror (the owner door)

`schema/lib.schema` declares `Configure(ConfigureRequest)`, `RegisterStore`/`RetireStore`, `SetRetention(RetentionOrder)`, `ObserveRegistry`, and `OrderRejected{StoreUnknown|StoreAlreadyRegistered|LedgerFault}`. `MirrorDaemonConfiguration` (storage path, working socket path+mode, meta socket path+mode, tailnet TCP `ListenAddress`) is BOTH the meta Configure payload and the daemon's single binary rkyv startup argument; `src/lib.rs:43-65` adds the binary archive surface (`from_binary_path`/`write_binary_file`) with a typed `ConfigurationArchiveError`. `ConfigurationWrite` is the deploy text-edge request `mirror-write-configuration` consumes. The schema comment and INTENT state plainly that retention is named-and-stored, NOT enforced (deferred by decision). 6 round-trip witnesses including the binary startup archive.

## mirror (the daemon)

**Planes.** `schema/nexus.schema` imports the wire roots from `signal-mirror` (single source, tb9h shape — the daemon emits no second copy of the wire types) and declares the feature catalog: `AppendDecision [(AcceptSuffix NovelSuffix) (AcknowledgeDuplicate AppendReceipt) (RefuseAppend AppendRejection)]` and `CheckpointDecision [(AcceptCheckpoint) (AcknowledgeCheckpoint) (RefuseCheckpoint)]`. `schema/sema.schema` declares the check/persist plane inputs and the mirror's OWN four record families (`StoredHead`/store-heads, `ReceivedEntry`/received-entries, `StoredCheckpoint`/checkpoint-artifacts, `RetentionSetting`/retention-rules) — the record-family-emission branch emits `family_identity` constants, the closed `RecordFamily` sum, per-family `sema_engine::TableDescriptor` constructors, and `RecordFamily::versioning_policy()` (store name `mirror` from the schema identity) at mirror/src/schema/sema.rs:709-805.

**Dogfooding.** `Store::open` (mirror/src/store.rs:212) opens the engine `with_versioning(RecordFamily::versioning_policy())`: every registration, received entry, checkpoint artifact, and retention row lands in the mirror's own versioned commit log and outbox — witnessed by `the_mirror_ledger_is_itself_a_versioned_store` (tests/daemon_logic.rs).

**Decisions.** `CheckedAppend::into_decision` (mirror/src/decision.rs:24) is the schema-declared decision: expected-head names the entry just before the suffix and must match the stored digest at that sequence; entries at/below the head must match stored digests (idempotent dedup — a fully-duplicate suffix acknowledges with the same head and writes nothing; a partially-novel suffix persists only the remainder); non-consecutive sequences are `SequenceGap`, chain breaks `HeadForked`, divergent re-sends `DigestMismatch`; rejections echo the current head for resync. `MirrorEngine` (src/engine.rs) implements the generated `NexusEngine` (hand-written execute loop driving CommandSemaRead/Write through its own SemaEngine impl) and `SemaEngine` (delegating to `Store`); meta orders run `handle_meta` against the same single writer.

**Listeners.** Generated Unix working + meta tiers per the agent exemplar (`impl ComponentDaemon for MirrorDaemon`, src/schema_daemon.rs; meta socket 0o600, meta decode component-owned, 1 MiB cap + 10 s read timeout). The tailnet TCP ingress (Spirit rj9y) is hand-wired: `MirrorService` (src/service.rs) is the kameo actor owning `MirrorEngine`; its own `on_start` binds `triad_runtime::TcpListenerDaemon` around its own `ActorRef` via `TailnetIngress` (`AsyncConnectionRuntime<TcpStream>`), using the SAME `LengthPrefixedCodec` and the SAME `signal-mirror` frame contract as the Unix working tier. `ComponentDaemon::Engine = ServiceLink` (a cloneable mailbox handle), so all three listeners serialise through ONE actor — the single writer is structural, and meta is structurally unreachable over TCP (the ingress decodes only the ordinary contract). TCP peers carry `PeerIdentity::Tcp`, witnessed in the arc test via the tested `observed_tcp_peers` state.

**fsync-then-ack.** Stated plainly in ARCHITECTURE.md: sema-engine commits through redb are durable at commit; `persist_suffix`/`persist_checkpoint` return only after the write transaction committed, and the `Appended` reply is composed strictly after — no second fsync layer invented. The two-transaction persist (entry rows, then head row) has a documented crash window healed by the shipper's idempotent re-send.

**CLIs.** `mirror` (working, `MIRROR_SOCKET`) and `meta-mirror` (meta, `MIRROR_META_SOCKET`), one NOTA argument each, binary frames to the daemon, NOTA reply rendering (src/client.rs); `mirror-write-configuration` is the deploy text edge; `mirror-daemon` takes exactly one binary rkyv configuration file (`MirrorDaemon::run_to_exit_code()`); text bins gated `required-features = ["nota-text"]`.

**The end-to-end arc witness** (tests/end_to_end_arc.rs, Spirit 29pb first cut): a fixture component store (sema-engine 0.4.0, hand-registered `thought` family) takes assert/mutate/checkpoint/assert/retract; the fixture `Shipper` reads `unshipped_outbox()`, loads the versioned entries, wraps each as a payload-blind envelope (payload = rkyv of the whole `VersionedCommitLogEntry`), ships over REAL loopback TCP frames to the RUNNING mirror service, and on `Appended` calls `acknowledge_mirror` — `store_durability()` and `durability_of(head)` flip to `ServerCommitted` and the outbox drains. The checkpoint ships as a `PortableCheckpoint{metadata, segments}` artifact. A full-history re-send acknowledges idempotently at the daemon level. A FRESH store fetches `Restore` (checkpoint + 2-entry suffix), imports through `begin_import`/`ImportSession`, and the normal query surface reads records identical to the source (`alpha revised`, `gamma third`, beta tombstoned), with matching commit cursors.

## designDecisions
WIRE ENTRY ENVELOPE: EntryEnvelope = (sequence CommitSequence, previous_digest (Optional EntryDigest), digest EntryDigest, payload PayloadBytes) where digests are fixed (Bytes 32) blake3 (x0ja) and payload is the component's own rkyv encoding of the whole VersionedCommitLogEntry. The envelope REPEATS the entry's chain coordinates beside the opaque bytes so the mirror validates the hash chain without decoding component types; only the owning component ever decodes the payload (the restorer round-trips it back into VersionedCommitLogEntry for ImportSession). CheckpointArtifact is the same shape for checkpoints: (store, checkpoint sequence, covered_end, checkpoint digest, artifact bytes) — covered_end rides the envelope so the mirror computes the restore suffix without decoding the artifact.

STORE IDENTITY -> ENGINE STORE NAMES: stores are wire-named by StoreName, the string form of sema-engine's VersionedStoreName (the component's VersioningPolicy store name; the arc witness uses "arc-witness" on both source and restored engines). Inside the mirror's own engine, rows are domain-keyed by "{store}/{sequence:020}" (zero-padded so lexicographic key order is numeric sequence order); head rows are keyed by the bare store name. The mirror's OWN store name is "mirror", derived by generation from the schema identity component (RecordFamily::STORE_NAME) — dogfooding falls out of the emission rather than being hand-asserted.

DEDUP INDEX CHOICE: no separate index. The received-entries family keyed by store+padded-sequence IS the dedup index: the SEMA read plane (CheckAppend) loads the KnownEntry digests for [first-1 ..= last] of the requested suffix in one key-range scan, and the Nexus AppendDecision compares wire digests against them. Re-sent entries at/below the head must equal stored digests (else DigestMismatch); only the strictly-novel remainder persists; a fully-duplicate suffix acknowledges with the unchanged head and writes nothing (witnessed by versioned-log length equality). Read-decide-write is atomic because every request from every transport serialises through the one MirrorService mailbox.

RETENTION PLACEHOLDER: meta-signal-mirror's RetentionOrder { scope (Store name | AllStores), rule (KeepEverything | KeepLatestCheckpoints count) } is persisted as the RetentionSetting family (keyed per-store or all-stores) — named, wire-typed, stored, and observable, but NOT enforced; enforcement (pruning received entries behind kept checkpoints) is deferred by decision and stated in the schema comment, INTENT.md, and ARCHITECTURE.md. Retiring a store retracts only its head row; history rows remain until retention enforcement lands.

TCP INGRESS HAND-WIRING: MirrorService (kameo actor owning MirrorEngine) is the one component runtime; the generated Unix daemon's ComponentDaemon::Engine is a cloneable ServiceLink into its mailbox, and the service's own on_start binds triad_runtime::TcpListenerDaemon around its own ActorRef (TailnetIngress: AsyncConnectionRuntime<TcpStream>, same LengthPrefixedCodec, same signal-mirror frame contract, one request frame per connection mirroring the generated working transport). This makes the single writer structural across three listeners and makes meta structurally unreachable over TCP (the ingress decodes only the ordinary contract). Hand-wiring was forced honestly: the emitted DaemonBinder constructs its EngineActor privately, so a second transport cannot share it. EMISSION FOLLOW-UP (named): a TcpListenerTier on NexusDaemonShape plus a shared-engine seam in the daemon emitter, so the generated binder can serve Unix + TCP from one engine actor; the mirror is the first consumer to copy from.

## Adversarial review

VERDICT: needs-fixes

## mustFix
- The documented crash-window self-heal is false and the store wedges permanently. persist_suffix writes two transactions (entry rows commit, then head mutate); ARCHITECTURE.md lines 69-73 and the store.rs comment claim a crash between them self-heals because 'the shipper's idempotent re-send dedups the rows and re-advances the head'. It does not: the novel-remainder filter in CheckedAppend::into_decision keeps every entry with sequence > head-row sequence (decision.rs:61-65), so a re-send re-asserts the already-persisted orphan rows, and sema-engine's Engine::commit rejects duplicate assert keys (pinned checkout 3593ffe src/engine.rs ~line 680: duplicate_assert_key). Empirically proven with an external witness crate (/tmp/mirror-crash-witness): the re-send returns MirrorFaulted("ledger storage: assert key already exists: received-entries/spirit/00000000000000000001") and every subsequent append for that store faults or gets SequenceGap forever — a liveness wedge at exactly the durability boundary the mirror exists for (0yx5 fsync-then-ack, 29pb). Fix direction: compute the novel remainder against the loaded known rows (digest-verified, divergence = DigestMismatch) and support a head-only re-advance when the entry remainder is empty (note Engine::commit also rejects empty commits, so persist_suffix must skip the entry commit in that case); then make the ARCHITECTURE claim true or delete it.
  AT: /git/github.com/LiGoldragon/mirror/src/store.rs:323-346 (persist_suffix + comment), /git/github.com/LiGoldragon/mirror/src/decision.rs:60-78 (novel filter), /git/github.com/LiGoldragon/mirror/ARCHITECTURE.md:69-73
- Retire-then-re-register permanently wedges a store name — same root cause, distinct documented flow. RetireStore retracts only the head row and 'history rows remain until retention enforcement lands' (store.rs:373-382); after re-registration the head is None, so only a genesis suffix (sequence 1) passes validation, and persisting it re-asserts the surviving entry rows -> duplicate-assert fault. Empirically witnessed in the same external test. Either refuse RegisterStore while history rows exist for the name (typed rejection), purge rows on retire, or fix via the known-rows dedup above so re-registration resumes the surviving chain.
  AT: /git/github.com/LiGoldragon/mirror/src/store.rs:373-382 (retire_store), /git/github.com/LiGoldragon/mirror/src/engine.rs:91-126 (register/retire meta path)

## advisory
- Dead schema-declared SEMA meta verbs: WriteInput::{RegisterStore,RetireStore,PersistRetention} and ReadInput::LoadRegistry are implemented in apply_inner/observe_inner (engine.rs:348-377,408-412) but nothing constructs them — handle_meta (engine.rs:69-148) drives Store methods directly, duplicating register/retire logic with slightly different validation (apply_inner's arms lack the already-registered/unknown checks). The Nexus/SEMA feature-catalog visibility rule is half-realized for the meta tier: declared verbs exist that the runtime never routes. Either route handle_meta through the declared plane or drop the dead variants.
- observed_tcp_peers grows unboundedly in production: every TCP request pushes a PeerIdentity into a Vec that is never drained (service.rs:42,120-122) — a test-witness surface leaking memory over the daemon's lifetime. Cap it, count it, or gate it.
- The generated DaemonBinder hard-codes the meta socket mode 0o600 (src/schema/daemon.rs:188) while MirrorDaemonConfiguration.meta_socket_mode rides the wire decoratively — two sources of truth; the config field is never honored. Either honor it or remove it from the contract (emission-level fix; named here so the TcpListenerTier follow-up also covers it).
- CLI and config error laundering: client.rs:54-61,82-91 and config.rs:101-111 stuff missing-argument, NOTA-parse, and configuration failures into Error::Io(std::io::Error::other(...)) strings, erasing typed reasons; configuration_writer.rs shows the correct typed-WriterError shape sitting beside them. Add typed variants (MissingArgument, NotaDecode, Configuration) to mirror's Error.
- signal-mirror declares thiserror = "2" but neither hand-written nor generated code uses it (generated SignalFrameError hand-implements Display/Error) — unused dependency.
- StoreName is unvalidated at the meta surface: the key scheme's isolation relies on store names not containing '/' (store.rs:33-36 comment), but RegisterStore accepts any string; a name carrying '/' breaks key-range isolation between stores. Refuse separator-bearing names with a typed rejection.
- Self-report inaccuracy: the design notes claim the mirror's own store name is "mirror" derived from RecordFamily::STORE_NAME, but the emitted constant is "mirror:sema" (src/schema/sema.rs:771); the dogfooding test witnesses log/outbox parity but never the store name. Code is fine; the claim was embellished.
- SocketMode::into_u32 uses a silent truncating `as` cast from u64 (meta-signal-mirror/src/lib.rs:31-35).
- The external witness crate at /tmp/mirror-crash-witness reproduces the mustFix in ~10 lines of driver; worth porting into tests/daemon_logic.rs once the fix lands (crash-window recovery and retire/re-register resumption are exactly the untested documented behaviors).

## disciplineFindings

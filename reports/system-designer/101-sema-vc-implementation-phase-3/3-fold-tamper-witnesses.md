## The two production fixes

**`Checkpoint::verify` count conflation** — the old shape reported every count mismatch as `SegmentMissing`, fabricating an all-zero digest when more segments were supplied than referenced (`metadata.segments.get(supplied_count)` out of bounds). Now a three-way match at `src/checkpoint.rs:504-521`: fewer-than-referenced returns `SegmentMissing` whose digest is `metadata.segments[supplied_count].digest()` — in bounds by construction, so always a real reference, never fabricated; more-than-referenced returns the new typed `Error::SegmentSurplus { referenced, supplied }` (`src/error.rs:100-103`). Both directions witnessed (below).

**`CommitLogOperation::new` typed-domain guard restored** — the constructor had been weakened to `impl Into<String>` to serve the `From<&VersionedLogOperation>` projection. Now `pub fn new(operation: SemaOperation, table: TableName, key: Option<RecordKey>)` (`src/log.rs:92`); all nine engine call sites already passed `TableName` and compile unchanged. The projection (`src/log.rs:118-130`, same module) constructs the struct literally from the logged string coordinate instead of widening the public surface; a doc comment states why.

## The witness suite — tests/tamper.rs (9 tests, all behavioral)

Doctoring uses two real attack surfaces, both with precedent in the repo (`tests/fold.rs:202` already opens the raw kernel):

- **Raw storage kernel**: the engine's internal tables redeclared in the test (`tests/tamper.rs:30-35`) and written through `sema::Sema` directly — the exact thing `StorageReader`'s read-only design forbids through any engine path.
- **`Splice`** (`tests/tamper.rs:86-128`): serialize a typed value, replace one equal-length byte run, decode back — the in-memory shape of on-disk tampering; it produces entries/metadata whose stored digests no longer recompute, which no public constructor can build.
- **`forged_replacement`** (`tests/tamper.rs:200`): an *internally valid* replacement entry — same chain coordinates, different payload, freshly computed digest — only its relation to the rest of the store betrays it.

## designDecisions
1) SegmentSurplus as its own variant rather than folding both directions into one count-mismatch error: SegmentMissing keeps its restore-oriented meaning ("fetch this digest") and now always names a real reference (index in bounds by construction); the surplus direction carries counts because a surplus segment has no reference to name. 2) CommitLogOperation::new takes TableName; the From<&VersionedLogOperation> projection constructs the struct literally inside src/log.rs instead of a pub(crate) string constructor — the narrowest surface that keeps the public boundary typed without widening anything for one in-module caller. 3) Tamper tests doctor through two surfaces only: the raw storage kernel (precedented in tests/fold.rs) for persisted-row forgery, and a byte-splice helper (a data-bearing Splice test noun, not free-function soup) for values whose stored digests must disagree with their fields — no test-only constructors were added to production code. 4) The two-pass forgery pattern (flip a field, then fix the outer digest using the error's own `computed` value) witnesses that inner digests (view digest, schema re-derivation) catch what a fixed-up outer digest cannot hide — and doubles as a witness that the error variants' carried fields are truthful. 5) tests/tamper.rs as one dedicated file rather than spreading across the per-module suites: the witnesses share the doctoring tooling and tell one adversarial story spanning fold, checkpoint, import, and outbox; the brief explicitly sanctioned this layout. 6) No INTENT.md edit: the brief quoted no Spirit records.

## advisory
- CommitLogOperation keeps table_name: String inside the rkyv-archived row (src/log.rs:87) while the public constructor now takes TableName — correct for this brief's no-redesign scope, and the conversion goes through impl From<TableName> for String (src/table.rs:29-33).

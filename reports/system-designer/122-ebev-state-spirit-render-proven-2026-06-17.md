## 1 — The mechanism is done (`spirit-render`)

`spirit::render::RenderCli` (bin `spirit-render`, `spirit/src/render.rs`) is the
`spirit.nota` generator the bead's refined mechanism describes — and it matches
that spec point-for-point:

- Input is one NOTA record `(Render ([<referents>] <Some dir | None>))`.
- It fetches the store **Marker** (`Input::Marker → MarkerReported`), then runs
  an **`Observe`** filtered to the named **referents**, **privacy exactly
  `Zero`** (public-only — the bead's hard requirement), certainty ≥ `Minimum`.
- Handles `RecordsObserved`, `RecordsStashed` (stash lookup), and the
  no-match → empty case.
- Writes `spirit.nota` with a header carrying generator-version, generated
  unix-timestamp, the **`spirit-marker`** (the staleness/version stamp, since
  records carry no per-record time), the privacy + certainty filters, and a
  `criome-auth-reference` slot (currently `None`).
- It is **tested** (`tests/spirit_render.rs`: request shape, empty-referent
  rejection, full document output) and **packaged** in the spirit flake
  (`--bin spirit-render`). `skills/spirit-cli.md` already documents it.

## 3 — What remains (pure rollout, no mechanism work)

1. **Deploy `spirit-render`.** It's in the flake but **not in the nix profile**
   (`which spirit-render` → not found), and its default socket
   (`/tmp/spirit.sock`) isn't where the live daemon listens. Deploying the bin
   (and aligning the socket default / `SPIRIT_SOCKET`) is a **system-operator /
   cluster-operator** task — agents can't regenerate `spirit.nota` until it's
   on the path.
2. **The contract-doc rewrite (`ebev` step 1).** `AGENTS.md` still says
   "that repo's `INTENT.md` FIRST … its absence is the first gap to fill" and
   names `INTENT.md` the most-important per-repo file. The intent skills
   (`intent-log.md`, `repo-intent.md`, `intent-manifestation.md`,
   `intent-maintenance.md`, `intent-clarification.md`) don't yet make Spirit
   canonical or point to `spirit.nota`. This is the keystone: it defines the
   convention everything else follows. The bead flags it **deliberate,
   high-blast-radius, cross-lane** — it sets the contract every agent reads.
3. **Per-repo generation + migration (gradual, opportunistic).** No
   `spirit.nota` is committed in any repo. The migration: when a repo is
   touched, confirm its `INTENT.md` intent is in Spirit (migrate gaps), generate
   `spirit.nota`, then slim/retire `INTENT.md` toward an architecture-adjacent
   summary or removal.

**jj discipline note (post-compaction self-observation).**
Several times in this session I edited files in the
working copy when `@` was on a published bookmark commit.
That mutates the bookmark's commit content, creating
local-vs-remote divergence (jj shows `(divergent)` and
`@origin (ahead by 1, behind by 1)`). Per `skills/jj.md` §
"The standard flow", the right discipline is `jj commit
-m '<msg>'` after each push so `@` advances to a new empty
child; the next edit then goes into a new commit, not the
published one. The pattern I used to recover (reset
bookmark to remote with `--allow-backwards`, `jj new
horizon-re-engineering@origin`, re-apply edits) works but
loses the working-copy state and requires re-doing the
edits — wasteful. Future edits should `jj commit` first to
detach `@` from the bookmark.

## Side notes

- **No-hashes preference.** The user wants commit hashes, store
  paths, and content-addressed identifiers kept out of chat
  output. Reference by branch + commit subject when needed.
- **Wired connection on this workstation is load-bearing.**
  Don't bring it down without permission — other agents on it
  need it. The `nmcli +ipv4.routes "<dst>/32 <wifi-gw>"` +
  `nmcli device reapply <iface>` recipe is the polkit-allowed
  per-destination route override that lets you test wifi-only
  paths without disturbing wired.
- **systemd-run for prometheus activations.** When activating
  on prometheus over an SSH path that depends on Wi-Fi (e.g.
  if hostapd will restart),
  `systemd-run --collect --unit=<name> --no-block <profile>/bin/switch-to-configuration <action>`
  detaches the activation from the SSH session so it survives
  the disconnect. Then poll `systemctl status <name>` after
  reconnect.

  `reports/system-assistant/` only. Building on
  system-specialist (or any other role's) reports happens by
  writing a *new* report in this lane that summarises and
  cites the original — never by editing the other role's
  report. Per `skills/system-assistant.md` §"Report from

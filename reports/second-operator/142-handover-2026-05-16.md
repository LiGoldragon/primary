   agent found ARCH edits in the persona working copy under parallel

   describe destinations for /184's four gaps and align with what

   report→ARCH cycle; operator-assistant ratify-and-commit; discard.
2. **P1 channel choreography handlers** (from /138). A minimal
   accept-all `ChoreographyAdjudicator` actor would bake silent
   defaults into four unanswered design questions: where the policy
   lives, who's authorised to grant channels, how grants gate the
   router, what retract means for in-flight messages.
3. **Kameo upstream `pre_notify_links`-after-`Self::drop()` hook**
   (from /138 + skill warning). Supervised `spawn_in_thread` releases
   `wait_for_shutdown` before `Self` drops, leaking redb file locks.
   Either upstream Kameo grows a post-drop hook, or actors adopt a
   close-then-confirm protocol. Worth filing upstream.
4. **Multi-thread parallel-restart test isolation** (from /138).
   Independent of #3: three `*_survives_process_restart` tests hang
   when run under `flavor = "multi_thread"` in parallel. Single-thread
   passes. Needs test isolation before any multi-thread switch.
5. **`terminal-cell-daemon` shared-lock drift** (from /140).
   `src/bin/terminal-cell-daemon.rs:196` uses
   `Arc<Mutex<TerminalSignalControlState>>` across socket-accept-loop
   tasks. Conflicts with `persona-terminal` ARCH §1.5's
   "`TerminalSignalControl` is a Kameo actor in persona-terminal"
   destination. Move signal-control to a Kameo actor in terminal-cell,
   or retire the surface in favor of persona-terminal.
6. **`signal-persona-system` doc-comment drift** (from /140).
   `SubscriptionRetracted` doc-comment at `src/lib.rs:277-279` still
   says "Path A reply variant ... not a request-only fire-and-forget
   op." Stale per /91 (it's both). Code is correct; doc-comment
   needs reconciliation on a future contract-side sweep.
7. **`persona-terminal-signal` CLI control/data migration** (from
   /141). After /189 + /197 split sockets and /141 updated dev-stack
   to use `--control-socket`/`--data-socket`, the smoke chain fails
   because the CLI client hasn't been updated for the split. Daemons
   launch fine from typed config; the breakage is in the smoke's
   client invocation.

# 533 — Deep situation report — frame and method

## What the psyche asked

*"Do a deep dive again into intent, into patterns. Do like a deep situation
report and bring up any important points that you see as crucial for me to
give an opinion or clarify on with proper context."*

This is **not** a re-run of the 532 meta-audit (pattern-of-patterns /
intent-of-intent synthesis). The deliverable here is **decision points** —
the genuine forks where psyche judgment is needed, each presented with
enough context that the psyche can decide *without opening the report*.
The discipline (`skills/intent-clarification.md`) applied at landscape
scale: scan intent + patterns, find where intent is unclear / contradicted
/ carried-uncertain / forked, and surface it.

## The five scan dimensions

1. **`1-intent-forks.md`** — the Spirit intent landscape: carried-uncertainty
   records (Medium/Low certainty), contradictions, open threads, the auditor
   role, anything awaiting psyche resolution. Where is intent *unsettled*?
2. **`2-engine-pattern-forks.md`** — the schema-derived / triad engine and
   code patterns: what is settled vs genuinely open (the emitter migration,
   the deployment gap, the structural macro node), and the forks within.
3. **`3-horizon-lojix-direction.md`** — the NEW horizon/lojix/cutover theme,
   deep: what it is, where it is headed, the sequencing forks, what it means
   for the existing triad engine. The biggest unknown.
4. **`4-discipline-and-content-forks.md`** — the workspace's own rules where
   they diverge or contest (sync-debt, report-kind vocabulary, lane-vs-topic
   maintenance authority, the `ui5d` quality discipline), and content-silt
   status after the operator sweep.
5. **`5-contrarian-blindspots.md`** — the completeness critic: what is NOT
   being surfaced that should be? The biggest unacknowledged risk; the fork
   the psyche would regret not being asked about.

The highest-numbered **`6-overview.md`** is the orchestrator's synthesis:
the ranked decision points, cross-dimension connections, and the
psyche-facing situation report.

## Method + boundaries

- Each scan agent writes a genuine report into this directory grounded in
  real evidence (record ids, file:line) and returns **verified decision
  points** — each adversarially checked for *"is this genuinely open, or
  does intent already settle it?"* so no fake questions reach the psyche.
- **Nothing destructive is executed by this audit.** It surfaces and
  contextualises; the psyche decides. (Contrast 532, which executed the
  safe already-flagged dedups.)
- The directory IS the meta-report; garbage-collected as one unit.

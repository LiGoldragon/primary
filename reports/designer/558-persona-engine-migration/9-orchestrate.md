### The two architectural truths worth surfacing

1. **The emitted shell already exists but is orphaned.** `build.rs:42-46` drives `GenerationDriver` + `NexusDaemonShape` and `write_or_check` keeps `src/schema/daemon.rs` fresh, so the `ComponentDaemon`/`DaemonEntry`/`GeneratedDaemonRuntime`/`AsyncMultiListenerDaemon` surface is present and compiles. Nothing in `src/` implements `ComponentDaemon` or calls `run_to_exit_code()`; `main.rs` instead instantiates the hand-written `OrchestrateDaemon`. Wiring is the bulk of axis-1 work, not generation.

2. **A third listener tier blocks a clean shell adoption.** orchestrate binds THREE sockets — ordinary, meta, and a private `upgrade` socket carrying `signal-version-handover` (`daemon.rs:242` `handle_upgrade_stream`, retiring public sockets on `HandoverFinalized`). The emitted `NexusDaemonShape`/`ComponentDaemon` only model Working + Meta tiers (`ListenerTier { Working, Meta }`, `daemon.rs:130-134` in the generated file). The handover tier must be expressed in the shape (a third/private tier) before the hand-written daemon can be deleted.

### Migration path for orchestrate

The execution plane is done; this is a daemon-shell and contract-emission job, not an executor-removal job. (1) Extend `schema-rust-next`'s `NexusDaemonShape` to carry the private upgrade/handover listener tier, then implement `ComponentDaemon` for orchestrate (Configuration/Engine/Error, `build_runtime` opening the sema store + service, `handle_working_input`, `handle_meta_connection`, and a handover handler), collapse `src/main.rs` to `<OrchestrateDaemon as DaemonEntry>::run_to_exit_code()`, and delete hand-written `src/daemon.rs` (MultiListenerDaemon/BoundedWorkers/OrchestrateRequestWorker). That single move also fixes axis-8 (the emitted `DaemonCommand` is binary-rkyv-only) and lets the daemon land on the actor substrate. (2) Introduce a real kameo state actor owning `OrchestrateTables` (ask-not-tell, RestartPolicy, redb via `spawn_blocking`), replacing the `Arc<OrchestrateService>`+`Mutex` sharing, to satisfy axis-7. (3) Finish the contract migration by moving the wire frame surface into the schema-next-emitted module and deleting `signal_channel!` from both `signal-orchestrate` and `meta-signal-orchestrate`. (4) Refresh ARCHITECTURE.md to drop the "actor tree" overclaim and describe the real (currently blocking) runtime plus the orphaned emitted shell. Effort: medium — the hard engine work is already paid; what remains is shell wiring gated on the third-tier daemon shape.

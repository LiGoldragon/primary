## router — migration assessment

**Role.** Message routing and delivery daemon: checks channel authorization, queues pending deliveries, attempts delivery through harness, commits typed delivery results, emits subscription deltas. Binds the working `router.sock` and a separate meta channel-policy socket. All three triad repos present (`router`, `signal-router`, `meta-signal-router`).

**One-line verdict.** Router is one of the furthest-along components on the *shell* and *boundary* axes (emitted daemon shell on the actor substrate, binary-only config, sema-engine storage, signal-executor fully gone including Cargo.lock) but has two real migration gaps left: the emitted nexus runner is unwired (execution still runs through the kameo ask-tree), and the working `signal-router` contract is still hand-written `signal_channel!`.

### The three terminal hazards (556), checked against router

Router shares all three hazards audit 556 found in terminal:

- **No RestartPolicy** for transient state actors (`grep RestartPolicy src` empty).
- **Leaked supervision thread** — `supervision.rs:82` spawns a blocking `UnixListener` loop into `_thread: JoinHandle<()>` that is never joined.
- **redb inline on the async handler** — `ChannelAuthority`'s `GrantChannel` handler synchronously calls `tables.insert_channel(..)` (`channel.rs:82`). Only `harness_delivery.rs:172` correctly uses `spawn_blocking`, and that is for the delivery effect, not storage.

### Migration path for router

Router has already cleared the hard structural axes — the daemon is the emitted shell on the actor-native substrate, config is binary-rkyv-only with no flags, storage is on sema-engine with emitted SEMA nouns, signal-executor is gone down to Cargo.lock, and the docs are unusually honest. The remaining work is two concrete migrations plus a hazard cleanup. First, **wire the emitted nexus runner**: implement `nexus::NexusEngine` and switch the daemon working path from `ask(ApplySignalMessage)` to `engine.execute(NexusWork::signal_arrived(..))`, demoting the kameo `RouterRuntime` tree from request-executor to the SEMA/effect backend the runner drives — this is the only axis where the generated infrastructure is present but unused. Second, **bring the working `signal-router` contract onto schema-next** (author the `.schema`, lower, emit, delete `signal_channel!`), which also lets the daemon's hand-written two-relation decode hook collapse onto schema roots. Alongside, apply the shared kameo fixes (RestartPolicy, off-handler redb, de-leak the supervision thread) and settle the `signal-engine-management` third-contract question at the workspace level. Effort is medium: no shell or storage rewrite, just runner wiring + one contract migration + the cross-component hazard pass.

# Skills corpus audit

## Skills corpus audit — findings and recommended actions

The corpus is **14,615 lines across 60 top-level skills + 5 rust sub-files**. Most of it is genuinely dense and well-factored — the NOTA cluster (design/comments/schema-docs), the intent cluster (log/maintenance/manifestation/clarification), and the rust index + sub-files all carry distinct jobs and cross-reference cleanly. The problems concentrate in two places: the **reporting cluster** (heavy duplication across 5 files) and a handful of **individually-overlong** files where the discipline could survive at half the length. No contradictions with ESSENCE.md or the AGENTS.md hard overrides were found; no stale references to retired mechanisms (legacy `owner-signal-*`/`intent/*.nota` mentions are all explicitly framed as migration-only).

### Ranked action list (by payoff)

**1. Collapse the reporting cluster — highest payoff.**
`reporting.md` (811) + `report-naming.md` (43) + `context-maintenance.md` (323) + `context-maintenance-deep.md` (77) + `workspace-update-report.md` (136) = ~1,390 lines for one discipline (where work-substance goes and how reports retire). The redundancy is concrete:
- The **filename convention, `-v2`/`-v3` iteration, and supersession-with-new-number** rules are stated fully in BOTH `reporting.md` §"Filename convention"/"Iterating" AND all of `report-naming.md`. `report-naming.md` is a strict subset of `reporting.md` — merge it in and delete it.
- The **meta-report-directory pattern** is defined in three files (`reporting.md` §"Meta-report directories", `context-maintenance.md`, and 5 mentions in `context-maintenance-deep.md`). Pick one home (`reporting.md`) and have the others reference it.
- The **drop/forward/migrate/keep** decision and the staleness-landing-gate appear in both `context-maintenance.md` and `context-maintenance-deep.md`.
- `reporting.md` itself is the single most winded file: the four "Psyche reports — …" subsections (show-the-code / narrative-voice / lean-vs-ratification) run ~90 lines and repeat the same "show real code, narrate, don't over-cite" point three ways. The "Questions to the user — paste the evidence" section (44 lines with a full wrong/right code example) substantially duplicates the same wrong/right-example treatment in `intent-clarification.md` §"How to ask" and `reporting.md` §"Human-facing references."
Recommendation: cut `reporting.md` to ~450 lines (fold the three psyche-report subsections into one; drop the duplicated cross-reference/inline-summary examples that `intent-log.md` §"Citing intent" already owns), absorb `report-naming.md` entirely, keep `context-maintenance.md` and `-deep.md` but de-duplicate the shared decision table to one definition.

**2. Merge `engine-analysis.md` (87) + `engine-report.md` (130).**
Two files, one discipline ("inspect how an engine currently works and write it up"). Both open with the identical readability test ("do the types name the work? schema names the interface, generated Rust names objects, handwritten code matches typed input→decide→typed output"), both carry the same five-state status taxonomy (hooked/stubbed/contract-only/conceptual/stale), both have a witness/proof-discipline section pointing at `architectural-truth-tests.md`, and both have near-identical visual rules. The only real distinction — "analysis" (deep passes) vs "report" (measurement+presentation+tooling) — is two sections of one skill, not two skills. Merge into one `engine-report.md` (~150 lines) with an "analysis passes" section and a "measurement + tooling" section; delete `engine-analysis.md`.

**3. Trim `designer.md` (582) — longest role file, ~40% sub-agent-dispatch mechanics.**
Roughly 200 lines (§"Parallel manifestation + audit pattern", §"Designer sub-agents land code witnesses", §"The designer-operator loop", §"Three-way convergence") are detailed sub-agent orchestration procedure. This is real, but it is procedure that belongs consolidated — much of it restates the meta-report-directory + worktree-branch + bead-handoff flow already in `reporting.md`, `feature-development.md`, and `main-next.md`. Cut to ~400 by compressing the loop/sub-agent sections to their load-bearing rules and cross-referencing the mechanics.

**4. `component-triad.md` (1,123) — longest file overall; mostly dense but over-budget.**
This is the single largest file and it is doing genuine work (the universal component shape + the runtime triad). It is not winded in the prose sense — nearly every paragraph carries an invariant. But at 1,123 lines it is two skills bolted together: the **repo triad / five invariants / one-argument rule** (lines 1-694) and the **runtime triad Signal/Nexus/SEMA planes** (lines 696-1,123). The runtime-triad half overlaps `actor-systems.md` §"schema-driven daemons" and `enum-contact-points.md`. Recommend splitting the runtime-triad half into its own `runtime-triad.md` (~430 lines) so each file is independently readable; the index already lists them as one entry but the psyche's "too long → confusing" complaint lands hardest here.

**5. `actor-systems.md` (628) — dense, keep, but two trimmable seams.**
Genuinely load-bearing (it is the thinking-discipline skill). The "Release before notify" shutdown-sequence material (~80 lines: numbered sequence, control-plane, terminal-outcome, how-to-apply table) is Kameo-lifecycle mechanics that arguably belong in `kameo.md` rather than the architecture-discipline skill — moving it would tighten the boundary the two files already claim ("architecture here, how-to-express-in-Kameo there"). Low-severity; flag don't force.

### What is genuinely tight (do not touch)
- `naming.md` (370), `abstractions.md` (247), `beauty.md` (83), `language-design.md` (282) — the craft core. Long where long is earned; the offender tables and diagnostic catalogues are reference material that pays per-lookup. `beauty.md` is exemplary: 83 lines carrying a complete diagnostic discipline.
- The NOTA trio (`nota-design` 255, `nota-comments` 146, `nota-schema-docs` 79) — three distinct jobs (design records / Why-records-in-code / pseudo-NOTA-for-docs), clean cross-refs, no overlap.
- The intent quartet (`intent-log` 373, `intent-maintenance` 153, `intent-manifestation` 149, `intent-clarification` 74) — record / sweep-and-supersede / push-to-guidance-files / ask-when-unclear. Each has a sharp boundary and they cross-reference rather than repeat. `intent-log.md` is the longest at 373 but the certainty ladder is reference material.
- The rust index + 5 sub-files — exemplary factoring; the index is 100 lines and delegates substance.

### Findings

**Daemons are real, not scaffolds.** `introspect/src/daemon.rs` fills the record-1488 `ComponentDaemon` escape hatches and drives a live `IntrospectionRoot` tree of **9 kameo actors** (`IntrospectionRoot`, `IntrospectionStore`, `TargetDirectory`, `QueryPlanner`, `ManagerClient`, `RouterClient`, `TerminalClient`, `NotaProjection`, `SupervisionPhase`). `RouterClient` is the one live peer; Manager/Terminal clients are honest scaffolds. `system/src/daemon.rs` drives **3 actors** (`SystemSupervisor`, `SupervisionPhase`, `FocusTracker`) and is intentionally paused — every unbuilt domain request returns a typed `SystemRequestUnimplemented { reason: NotBuiltYet }`. No `todo!`/`unimplemented!`/panic stubs anywhere in the cluster.

**Encoding-drift flag (the one thing to act on).** introspect, signal-introspect, system, and signal-system all pin **nota-next `ae5c25c`** — the exact rev schema-rust-next deliberately holds to isolate the encoding bump (SAFE, 4 behind `d8862b6`). But **`meta-signal-introspect` and `meta-signal-system` pin `16493c8`** ("render bracket strings with lossless pipe fences"), which is one commit *past* `ae5c25c` and an ancestor of `d8862b6` — i.e. inside the encoding-bump lineage, 3 behind HEAD. Both meta crates were created Jun 9 and grabbed a fresher nota-next than the emitter was built against. Worse, each meta crate's `Cargo.lock` resolves its `signal-introspect`/`signal-system` dependency forward to `16493c8` too, so the contract is encoded against a newer codec than its own working-signal dependency's lock declares. Recommendation: pull both meta crates *back* to `ae5c25c` to match the rest of the cluster, or push the whole cluster *forward* to `d8862b6` with a regenerated emitter — but don't leave the meta crates straddling the bump.

**Generated code predates today's grammar.** Both `src/schema/daemon.rs` files (260 LOC each) are `@generated by schema-rust-next` at pin `7282446`, **8 commits behind** HEAD `eca4028`, and predate the terse-form grammar (`52ro`/`yp29`/`qz6j`/`lm84`) and the Bytes/hash codecs that landed today. A coordinated regen across both daemons would land the new grammar uniformly and is the cleanest path.

**Schema files have decayed into stale documentation.** Every `*.concept.schema` in the cluster is written in the OLD verbose self-tag form (`Tap [(Observation Observation)]`, `Status [Concept Active Retired]`), untouched since "add v0.1 concept schema", and **consumed by nothing** — `build.rs` emits from a hardcoded Rust `NexusDaemonShape`, reading the `.schema` files only as `rerun-if-changed` markers. The concept schemas in `signal-introspect`/`signal-system` are byte-identical copies of their daemon repos'. `system` additionally carries a newer split `nexus.schema` (`SignalArrived`/`NexusWork`/`NexusAction` over `SystemToken`) that points the right direction but is still not the true codegen source. The live source of truth is the typed Rust; the `*.concept.schema` files are stale cruft to retire (or regenerate from the types). The two meta crates have **no schema dir at all** — the cleanest posture, contract-as-Rust via `signal_channel!`.

**Lock-file note.** `flake.lock` in the four flake-carrying repos pins only the Nix toolchain (`crane`/`fenix`/`nixpkgs`/`rust-analyzer-src`) — no foundation-crate inputs — so there is **no Cargo.lock-vs-flake.lock disagreement** to reconcile; foundation pins live exclusively in `Cargo.lock`. The two meta crates have no `flake.lock` at all. signal-system is the only repo lagging `signal-frame` (`8b128d3`, 1 behind `166bda8`).

### Fake NOTA (1 real + 1 borderline)

- **signal-system `src/lib.rs:78`** — `SystemTarget::to_nota` hand-writes `format!("(NiriWindow {})", window_id.to_nota())`. The inner `window_id.to_nota()` delegation is correct; the `(Variant …)` wrapper is hand-assembled where `#[derive(NotaEncode)]` (with `52ro` self-tag) would emit it. A single-variant enum that should derive the codec like every sibling record in the file. All `from_nota_block` impls in this crate correctly use the real nota-next parser (`expect_children`/`parse_integer`) and are **not** flagged.
- **introspect `src/surface.rs:71`** (borderline) — `Output::to_nota` delegates each payload to its own `.to_nota()` but hand-wraps the variant head via `wrap(head, payload) -> format!("({head} {payload})")`. The code comment explains *why*: the `signal_channel!`-generated `IntrospectionReply` enum doesn't carry `NotaEncode`, so projection can't derive. This is a **derive gap in the emitter**, not author cheating — the right fix is to make the generated reply enum carry the codec. (`wrap` is a method on `Output`, so it's not a free-fn violation.)

All scalar-newtype `to_nota` impls across the cluster (`self.0.to_string()` for integer leaves) and all `from_nota_block` parser calls are legitimate and were **not** flagged.

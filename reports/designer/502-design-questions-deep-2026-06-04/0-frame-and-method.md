# 502 - Design questions deep

## The directive

The psyche, 2026-06-04: make a full psyche report with my most important
questions — the admission-policy one (does route/identifier minting +
validation become a SignalEngine trait method?) being one of them. Explain it
more (including the time-horizon, "is this in one year?"). Give lots of clear
real code, what happens, what I suggest, what I think the new code would do,
and **short graphs with short identifiers** so nothing is cut off and they are
readable (captured at record 2566).

## Method

Four agents, one per question; each gathers the REAL code today (verbatim,
file:line), drafts the proposed/new code, says what happens and what is
suggested and what the new code would do, and makes ONE short mermaid (5 nodes
max, short labels). The orchestrator writes the psyche report 503 in the
designer voice with the code and visuals consolidated.

| Entry | Question |
|---|---|
| 1 | Admission policy as a SignalEngine trait method (the centerpiece, deepest) |
| 2 | Runner concurrency — sequential single-flight vs generated actor mailbox |
| 3 | SymbolPath — flat vs structured, both in code |
| 4 | The cutover mechanism — the undesigned switch (gate, migration, rollback) |

The psyche report 503 carries the consolidated code, the suggestions, and the
short visuals.

## Synthesis — Scratchpad Pollution in the Spirit Intent Store

### The store is moderately polluted, concentrated, and treatable

Of 1706 public records, 1348 (79%) are clean durable intent. 184 (10.8%) are confident removal candidates and 174 (10.2%) are borderline. So roughly one record in five wants the psyche's eyes, but only about one in nine is a confident removal. This is not a rotten store — it is a healthy intent layer with a recognizable, bounded contamination that maps onto where the heaviest restructuring work has been happening.

### How agents polluted it — the recurring shapes

The dominant failure is the **working-instruction in a Decision costume** (168 flagged). The tell is structural: the record names a *target artifact* (a repo, a version, a tool) plus an *action verb* — `port orchestrate`, `migrate signal_channel! to schema-next`, `create a cloud tool for Cloudflare DNS`, `start using spirit v0.1.1 after the DB update`. These die the moment the task lands. They wear the Decision Kind because "we will do X" and "the system should be X" are easy to confuse, but only the second survives task-erasure.

Close behind: **status-snapshots frozen as intent** (58) — `old macro infra retires`, `DJI keep-alive is currently broken`, `remaining string-based codegen is debt` — each a photograph of a moment in the code that expires on the next commit; and **migration narrative wrapped around a real decision** (the engine of the borderline count). The most common mixed record carries a genuine durable kernel — *signal-frame is the home repo*, *emit to `src/schema/` for grep-ability*, *copy+migrate not shared redb* — buried inside operator-correction status and per-commit consumer instructions. The willed shape is real; the record is mostly task-state by volume.

Three smaller but sharper classes round it out. **Production-framing** (`signal_channel! deprecated; migrate to schema-next`) is a category error the workspace contract explicitly forbids — nothing has shipped, so `deprecated/retire/legacy-compat` has no referent. **Agent-conclusions** (16) are the cleanest violations of the psyche/agent boundary: bare ratification tokens (`Migration split affirmed`), self-corrections with proof-green commit hashes, and records literally labeled `pending psyche confirmation` — an agent's finding logged as the human's will. And a thin layer of **self-superseding or conflicting pairs** (dppt vs dnvs on what certainty means for removal; p4dj retired by op4b) that need reconciliation rather than deletion.

### What it says about agent behavior

Pollution is not spread evenly — it concentrates on the **active migration fronts**: signal-frame, the schema-derived emitter, the Spirit cutover, triad-engine. When a large restructuring is in flight, agents reach for Spirit to record progress, hand-off slices, and next-steps, because in the moment those feel like the important durable facts. The Gate test ("a working order is not intent; if the statement dies when the task is erased, it is task state") was under-applied precisely where the work was hottest. The store is clean in stable domains and dirty in churning ones — which means the cure is disciplinary, not architectural.

### Recommendation — tiered, psyche-gated, and prevention-first

Never a blanket purge. Three tiers by judgment cost:

- **Tier 1 — confirm-and-remove (low judgment):** the 16 agent-conclusions, 3 production-framing records, and pure bookkeeping pointers (lane-rename, worktree-path, single-report pointers). Present as a flat batch; the psyche confirms.
- **Tier 2 — rewrite-then-remove (the 58 status-snapshots, 30 impl-detail records):** extract the durable kernel into a tight reworded Decision, then drop the task-state wrapper. Delete only the costume, keep the will.
- **Tier 3 — reconcile (duplicates / conflicts):** the psyche picks the surviving record and retires the other; removal here means choosing which truth wins.

The 174 borderlines stay until the psyche reviews them — default KEEP, because the conservative classification means many hide a genuine willed shape.

Prevention is the real fix, and most of it is already in the contract. Enforce the existing four-outcome Spirit gate on every prompt — the failure is agents skipping it on migration work. Add one mechanical pre-capture check: *target-artifact + action-verb = presumptive working-instruction → route to a report or ARCHITECTURE.md, not Spirit.* Status and migration mechanics belong in `reports/<role>/` and the repo's `ARCHITECTURE.md` by the existing report-vs-chat rule. Agent findings are never Spirit records until the psyche ratifies them — the "agent messages are not psyche" override is the literal bar. Finally, the coming **auditor lane** should run exactly this classification on a cadence, auto-proposing removals for psyche confirmation, so the store self-cleans instead of accreting the next migration's worth of scratchpad.

# Synthesis & verdict

Yesterday's designer work was substantively sound and the headline claims hold up: every "landed" SHA verifies at `HEAD = main = origin/main` with a clean tree, the two reuse goals (single-sourcing the criome decision mapping via `CriomeVerdict`, and reusing `ObservationModel`/`RenderNota` in the egui shell) were achieved correctly, and three of the four governing Spirit records (7x5z, p43g, pviw) are faithfully realized in code. The work is honest about itself where it counts — `mentci-lib/INTENT.md` even retracts the earlier "nexus-daemon dual pair" framing — and no report claims something is on main that is actually missing. But the polish does not reach the runtime or the keystone deliverable: the criome verdict-by-slot path (the central t00s control surface) is two dead building blocks with no plumbing between them and cannot be wired as built because the contract doesn't carry the slot, while the orchestrate registry, though merged, runs against a still-live schema-2 daemon and so is inert in production. The recurring quality smell is decorative discipline — a typed `Error` module nobody returns, public methods (`receive`) and commands (`SubmitCriomeVerdict`) with no producer, and contract types duplicated across four-to-eight schemas under a "deferred import" premise that `signal-standard` already invalidated. None of this is alarming for a single day's design wave, but the gap between "landed on main" and "live and load-bearing" is the theme, and it is wide enough to name plainly.

## What's strong

- **Genuine dual reuse, not nominal.** `mentci-lib` is consumed by both the `mentci` daemon (`criome_bridge.rs` calls `CriomeVerdict::from_decision`) and the `mentci-egui` client (holds a real `ObservationModel`, renders through the shared `RenderNota`) — the load-bearing claim of record 7x5z is true on disk, and the "when convenient" criome-mapping handoff is closed.
- **The criome decider model is correctly shaped.** `criome/src/actors/root.rs::evaluate_authorization` makes criome the decider (not a signature-verifier), parks client-approval requests by `AuthorizationRequestSlot`, and single-sources the verdict path by blocking `RejectAuthorization` in ClientApproval mode — faithful to p43g, and the `EscalateToPsyche` dead-letter (pviw) is a closed typed variant that is correctly inert.
- **Infrastructure-minted facts stay infrastructure-minted.** The worktree registry re-derives `last_activity` and `PushedState` from `jj` rather than trusting agent-supplied values — a good fidelity touch in `orchestrate/src/worktree.rs`.
- **Honest, current per-repo INTENT.md across most of the wave.** `mentci-lib`, `mentci`, and `criome` all refreshed INTENT.md the same day, including an explicit retraction of a superseded design framing.
- **Clean integration hygiene.** Prototype `[patch]` scaffolds removed, codegen pins resolved consistently across all orchestrate repos, preservation bookmarks pushed, and all six merged feature bookmarks retired.

## What needs work

1. **`mentci-lib` — the keystone verdict-by-slot path is unwired and unwireable as built (medium).** `Cmd::SubmitCriomeVerdict` is constructed nowhere and `CriomeVerdict` is never produced in the model; `on_user_event(AnswerQuestion)` routes every verdict to the Mentci socket with no criome branch. Worse, it cannot be wired today because `signal-mentci`'s `ApprovalSource::CriomeEscalation` carries no `AuthorizationRequestSlot`. Decide: close the contract cross-import and drive the criome branch, or delete the dead `Cmd::SubmitCriomeVerdict` and document `CriomeVerdict` as a pure mapping the daemon imports — do not ship both half-built.
2. **`signal-standard` regression — contract types duplicated and already forking (medium).** `signal-standard` exists as a built crate, yet `ComponentKind` is re-declared in four schemas, `SocketPath`/`StandardSocket` in eight, and yesterday's `meta-signal-mentci` narrowed `StandardSocket` to a single-field struct while the canonical is an enum with a network case — the same name, different shapes, which is exactly how a "deferred import" becomes a silent fork. Zero repos `use signal_standard`.
3. **`mentci-lib` — silent error-swallowing behind a decorative `Error` module (medium aggregate).** The entire `error.rs` (Error/Result/variants written precisely for the unknown-question / unknown-token / unheld-token cases) is never returned by any method; those paths instead return empty `Vec` indistinguishable from success. A control surface that drops a psyche's verdict because the queue moved, with zero signal, is a correctness hazard. `RetractObservation` compounds this by emitting the Cmd without clearing the local slot, leaving a retracted token that still passes the fold guard.
4. **`signal-mentci` — the focus-delta readers have zero test coverage (medium).** The six public readers whose entire reason for existing was to let `mentci-lib` project state it didn't build are never exercised by a test; `pending_questions()` (the only real logic, a 4-arm match with the FullProjection-vs-PendingQuestionsProjection equivalence) can silently regress.
5. **`orchestrate` (eh5a) — lifecycle and GC are stubs, INTENT.md stale (medium).** `WorktreeStatus [Active Merged Archived Recycled]` exists in the wire but every code path sets `Active`; there is no merge/archive/recycle order, and `worktrees.nota` is a passive projection nothing reads to GC. `orchestrate/INTENT.md` was not refreshed for the registry work — the one repo in the wave that shipped landing intent to main without updating its INTENT.md, against the same-branch rule.
6. **Contract-fidelity bugs in the generated wire layer (high).** Two are real correctness defects, not polish: `signal-orchestrate`'s generated mirror and canonical struct disagree on the `Worktree` status field name (`worktree_status` vs `status`, root cause `schema/lib.schema:45`), and `meta-signal-orchestrate` carries a **hand-rolled NOTA codec** for `WorktreeIndexRefreshed` that both violates the no-hand-rolled-parsers rule and disagrees with its own generated schema. Compounding both: every contract delta this wave shipped with **zero round-trip test coverage** in repos whose only job is wire fidelity (`signal-orchestrate` worktree vocab, `meta-signal-orchestrate` four types, `signal-mentci` six readers, `signal-criome` peer delta) — the one class of bug these crates exist to prevent is the one untested.
7. **Lower-severity cleanups.** Dead `ComponentLabel` newtype with a hand-written accessor (`signal-mentci`); stale `[patch]` "Scaffold note" and `InterfaceStateSnapshot` doc references describing removed shapes; unmerged `ApprovalModel::receive` fan-out path no event drives; per-repo copy-paste of the `string_accessor!` macro and channel-alias septet that should be schema-emitted; and `PrincipalId` violating the spell-identifiers-as-full-words override in `signal-criome`.

# 661/4 — The prototype (green) and the adversarial verification

## The prototype — composition proven green on real code

A designer prototype on the `reaction-expand` worktrees (uncommitted, not pushed — the report
`660` posture). It implements the smallest provable extension and proves both halves: a composed
body generates correct Rust, and the boundary fires as a typed error.

**Implemented:**
- `Expression::MethodCall(Box<Expression>, Name, Vec<Expression>)` — a *positional tuple*
  variant (not the design's struct sketch: `NotaDecode`/`NotaEncode` reject named-field enum
  variants, and NOTA is positional anyway).
- A data-bearing `ComposablePrimitive` enum (`Payload`, `IntoPayload`, `AsStr`) with
  `resolve(&Name) -> Option<Self>` — the closed alphabet, made explicit.
- `Expression::to_rust` now returns `Result<String, SchemaError>`; the `MethodCall` arm resolves
  the head against the alphabet and on miss returns the new typed
  `SchemaError::UnresolvedComposition { method, receiver }`. **The three panics are gone.**
- The `(call <receiver> <method> <args>…)` NOTA reader form (parse + lower + round-trip), via the
  existing nota-codec — no hand-rolled parser.
- The emitter dispatch: a rejected body emits a `compile_error!` carrying the typed message, so a
  business-logic body declared as data fails loudly at the generated-code boundary too.

**The schema body and the generated Rust.** Real method `ConfigurationPath::as_str`
(`signal-spirit/src/lib.rs:26-28`, hand-written `self.payload().as_str()`), declared as data:

```
ConfigurationPathAsStr {| Composed ConfigurationPath [ (as_str (call (call self payload) as_str)) ] |}
```

(Receiver is bare `self`, not `(self)` — the reader recognizes `self` as a bare atom; this
corrects the design's illustrative `(call (self) …)`.) Generated by the real emitter:

```rust
impl ConfigurationPath {
    pub fn as_str(&self) -> &str { self.payload().as_str() }
}
```

**Byte-identical** to the hand-written body. A depth-2 composition: outer `as_str` over receiver
`self.payload()`, bottoming out in the `String` leaf.

**Exact test results** (`composition_demo`, 7 tests, driving real parse → `to_rust()` → real
emitter → `include!` → compile → run):

```
cargo test --test composition_demo   →  7 passed; 0 failed
cargo test  (schema-next, all bins)   →  171 passed; 0 failed
cargo test  (schema-rust-next, all)   →  105 passed; 0 failed
cargo clippy --all-targets  (both)    →  exit 0, no warnings
```

The negative tests prove the boundary fires as a typed error, not a panic and not silent bad
output: `(call self keywords)` → `UnresolvedComposition{method:"keywords"}` (a real
business-logic method); and `self.payload().keywords()` rejects at the *first* non-primitive.

## The adversarial verification — 1 survived, 3 refuted

Four skeptics, each prompted to refute (default-refuted-if-uncertain). The verdicts:

| Claim | Verdict | What it means |
|---|---|---|
| Deletes **meaningful** hand-wiring | **SURVIVED** (high) | Could not be refuted against live code — see `2`. The win is real. |
| Not an **arbitrary expression compiler** | **REFUTED** (high) | True for `P(T)`; the **leaf registry leaks** (Vocab B). → forced the two-vocabulary split. |
| **Deterministic** resolution by object type | **REFUTED** (high) | True for the schema-shape wall; breaks at std types (autoderef `String`→`str`; `Option::map` higher-order). |
| **Terminates + compiles correct** | **REFUTED** (high) | Termination holds; *correctness* fails on borrow/move and variant-discard (see `3` hazards). |

### Why "refuted" here is a feature, not a failure

The survived claim is the one that matters — *the model deletes meaningful, recurring code.*
The three refutations did not break the thesis; they **bounded it honestly**:

- **The leaf-registry leak** (claim 2): `as_str` (accepted) and `trim`/`to_lowercase`/
  `wrapping_add` (rejected) are structurally identical bodies — `self.payload().<str-method>()`.
  The boundary between them is a maintained list, not a structural closure. → The closure claim
  must be scoped to Vocabulary A; Vocabulary B is a curated allowlist, presented as such (`3`).
- **Resolution at std types** (claim 3): `Description::trim = self.payload().trim()` reaches
  `str::trim` via Rust autoderef (`String`→`str`); `Option::map(Type::as_str)` is higher-order
  application. Modeling these *is* compiler front-end work. → The prototype sidesteps it
  correctly: the schema-level check is just "is this name in the closed alphabet?" (deterministic);
  actual Rust method resolution is left to `rustc`, and wrong compositions fail to compile (caught).
- **Correctness beyond the proven slice** (claim 4): `Self::new(x.payload())` is a borrow/move
  type error for non-`Copy` payloads; `Input::State(_) => Self::State` needs payload-discard. →
  Both are deferred past the proven `self`-only slice and are now explicit gates (`3`, `5`).

The prototype's own design already embodies the honest answer to the leak: a **closed enum
alphabet** (3 entries), rejecting everything else with a typed error. The correction is not to
the prototype's mechanism but to the *claim* about it — the alphabet is curated, not
shape-derived, and that is fine as long as it is named honestly and kept minimal.

ID kfqa | Decision | certainty=High importance=Minimum | domains=Technology.Software.Programming.Parsing | referents=schema,nota,parser,nota-body-decode,namespace
DESC: The schema reader is multi-pass NOTA-first: one universal NOTA parser produces a generic NotaValue tree, then context-sensitive macro passes interpret values per their NodeDefinitionPoint position, then assembly produces canonical schema — one syntax (NOTA), one parser, multiple semantic passes. NOTA parsing breaks down by object via positional ranges: each block tracks its source span and carries shape predicates (is_square_bracket_block / is_parenthesis_block / is_brace_block; holds_single/two/N_root_objects; recursive predicates like second_root_object_is_a_square_bracket_object), and each root object inside a block is itself a typed/positioned block, so the recursive parsing chain IS the chain of block-level queries. Schema parsing first identifies NOTA delimiters and their roles (parens for enum/variant, square brackets for vector-like sequences and string-derived token vectors, curly braces for name-value namespace maps), then runs passes per source block: pass 1 finds identifier uses and locations, pass 2 registers inline declarations into the namespace and resolves references by lookup — supporting bare-PascalCase-or-explicit-Variant-Payload shorthand, with the source representation (not lowered fields) telling whether each identifier defines, references, or both. NotaBodyDecode is the semantic decode entry point: document/block decode strip their wrapper context and delegate to the body decoder. After structural parsing matches a file body or delimited object, the next semantic step receives the matched body's inner object stream rather than the outer delimiter wrapper, and the expected type decides whether the body reads as struct, vector, enum variant, or other value. Schema source needs its own typed node model before assembled schema — a streamlined tree of typed structural-macro nodes, namespaces, paths, fields, variants, references representing syntax directly (including inline PascalCase local declarations and self-named field shorthand) — and schema can act as its own codec so shorthand round-trips as typed data rather than being a one-way parser convenience. Block reassembly concatenates blocks one-after-another; parsed blocks are first-class units re-emitted in order without nesting reconstruction.

DESC: Short-header / signal-type structure: a 64-bit micro-message header (1 root enum + 7 sub-enums) discriminates any object in the namespace in constant time, emitted by every signal contract macro; byte 0 is the ROOT VERB (most universal beingness — Help/Query/Message), bytes 1-7 are the verb-namespace sub-variants. Integers are typed values, not sub-enum kinds. Three sizing tiers: (1) 64-bit micro header, (2) 64-byte summary message (hand-implemented from natural summary fields, basis for a subscription tier), (3) full unrestricted rkyv message.

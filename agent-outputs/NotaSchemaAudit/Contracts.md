# NOTA Schema Contract Audit — Manifested Anti-Pattern Inventory

## Task and scope

Read-only sweep of the GENERATED contract/schema Rust across the workspace for
manifested non-canonical NOTA anti-patterns, classified per instance. No files
were edited. Scope: every `@generated by schema-rust` file under
`/git/github.com/LiGoldragon/`.

Doctrine baseline loaded: `nota-design`, `structural-forms`,
`rust-storage-and-wire`. Canonical rule applied: NOTA records are positional;
enum variants are named; an optional value encodes as a self-contained block
(`None` or `(Some x)`) that always occupies exactly one slot. A codec that
DROPS the slot / `Some`/`None` wrapper and emits a shortened bare variant atom
is the deviation.

## Coverage

Discovery: `grep -rl '@generated by schema-rust'` → 91 generated files across
all `signal-*`, `meta-signal-*`, and component repos (agent, cloud,
domain-criome, harness, introspect, lojix, message, mind, mirror, orchestrate,
persona, repository-ledger, router, spirit, system, terminal, terminal-cell,
upgrade, skills), plus the generator's own fixtures in `schema-rust-next`.

Sweeps run against the definitive 91-file set:
- variant/tuple `Option` payloads: `^\s*[A-Z]\w*\((Option|std::option::Option|core::option::Option|Vec<Option)` (catches multi-line/qualified forms too);
- hand-written codecs: `impl nota::NotaBody(Decode|Encode)`, plus `NotaDecode`/`NotaEncode`/`Display`/`FromStr`;
- implicit defaults / dropped values: `unwrap_or`, `::default()`, `Default::default`.

Nothing capped or skipped within the generated set. The generic `Option<Inner>`
codec was read at `nota-next/src/codec.rs:939-958` to fix the canonical baseline.

## Headline result

- **True positional/variant anti-pattern (enum tuple-variant `Option<T>` payload
  with a hand-written bare/shortened-form codec): 13 variants, in EXACTLY 2
  crates** — `signal-spirit` (12) and `signal-mentci` (1).
- The **"exactly two crates" claim HOLDS.** No third crate carries this defect.
- The 13 defect variants are backed by 3 pairs of hand-written
  `NotaBodyDecode`/`NotaBodyEncode` impls (the mechanism of the defect); these
  are the only hand-written NOTA-body codecs in the entire generated set.
- The ~50 `struct X(Option<T>)` newtypes are the legitimate named-optional-field
  pattern (domain-typed) and are OUT OF SCOPE.
- Display/FromStr impls are canonical projections or thiserror-style error
  messages, not NOTA hand-rolls. OUT OF SCOPE.

## Why the baseline separates the two shapes

`nota-next/src/codec.rs:948-957`: the generic `Option` encode is
`Some(x) → "(Some x)"`, `None → "None"`. Both are self-contained single blocks.
So an `Option<T>` reached through the generic path (whether a bare field, a
newtype-wrapped field, or even a generic-path variant payload) always fills
exactly one positional slot and never collapses siblings — canonical.

The defect enums instead carry HAND-WRITTEN codecs (emitted by the generator
only when the schema source declares an optional VARIANT payload) that:
- decode: accept a bare variant atom and inject `None`
  (e.g. `signal-mentci/src/schema/lib.rs:469` `"NotificationProjection" => Ok(Self::NotificationProjection(None))`);
- encode: DROP the payload slot when `None`, emitting the shortened bare atom
  (`signal-mentci/src/schema/lib.rs:564-570`, `signal-spirit/src/schema/domain.rs:816-821`);
- encode: also drop the `Some(...)` wrapper on the present case.

That shortened/bare form is exactly the "emits a shortened form" + "accepts a
bare form where a positional component should appear" + "hand-rolled codec that
does not use the generic path" anti-pattern trio.

## Implicit-default / dropped-value sweep

No NOTA value-drops or field defaults in decode. The only `::default()` /
`unwrap_or` hits are transport/runtime plane: `LengthPrefixedCodec::default()`
(wire framing), `triad_runtime::ContinuationLimit::default()` (runtime config),
`EmittedSubscriptions::default()` (daemon state init). None touch NOTA field
decoding.

## Generator note (root cause, provisional)

`schema-rust-next` fixtures contain zero enum-variant `Option` payloads and 3
legitimate newtype-`Option` structs. The generator emits the shortened-form
hand-written codec ONLY when a schema source declares an OPTIONAL VARIANT
PAYLOAD. So the 13 defects trace to two schema sources (spirit domain taxonomy,
mentci interface projection). Durable fix belongs in those two schemas (split
into named variants) and/or a generator guard that rejects optional variant
payloads. Recorded as a recommendation, not accepted doctrine.

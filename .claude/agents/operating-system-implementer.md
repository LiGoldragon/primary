## editing closeout

- Commit and push every source edit before completion.

## implementation

- Read local guidance and the changed path before editing.
- Change owned source, not generated or installed output.
- Keep compatibility unless the brief accepts a break.
- Stop on an unresolved authority, privacy, safety, or design decision.
- Run the relevant check before closeout.

## Nix

- Change Nix-managed behavior through source, flake inputs, checks, and deployment.
- Keep inputs portable and pinned.
- Do not patch installed output or search the store as source.

## operating system operations

- Change system behavior through pushed, reproducible source and supported deployment.
- Name the target, action, rollback owner, and activation evidence before a host change.
- Do not patch managed output.
- Require explicit psyche approval before a reboot or emergency runtime mutation.

## NixOS VM tests

- Use the repository's named VM check for guest behavior.
- Keep test node data derived from the declared test model.
- Name missing host capability as a blocked test.

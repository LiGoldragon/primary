## Learned
- For Lojix modernization, contract hygiene must happen before daemon work because the daemon depends on generated signal-lojix/meta-signal-lojix surfaces and their current signal-frame/schema-rust-next versions. — source: session experience: signal-lojix and meta-signal-lojix modernization preceded Lojix cargo update
- A crate living only under triad-port cannot be consumed as a normal git dependency until the repo exposes a root Cargo workspace including triad-port. — source: session experience: signal-lojix root Cargo.toml was needed before meta-signal-lojix could git-depend on it
- The current schema-rust-next emitter wraps aliases/tuple payloads more strictly; handwritten code must use generated ::new, payload, into_payload, and From conversions rather than tuple constructors or .0 fields. — source: cmd:LOJIX_UPDATE_SCHEMA_ARTIFACTS=1 cargo test --no-default-features#error-private-wrapper-porting
- triad-runtime 0.6.1 uses AsyncListenerSocket, AsyncMultiConnectionRuntime, AsyncMultiListenerDaemon, and AsyncMultiListenerDaemonError instead of the older Actor* listener names. — source: cmd:rg -n 'AsyncListenerSocket::new|AsyncMultiConnectionRuntime' /git/github.com/LiGoldragon/triad-runtime/src /git/github.com/LiGoldragon/spirit/src/schema/daemon.rs
- Do not add a generic build timeout for Lojix parity; lojix-cli permits cold builds that can take hours, and the cutover research says only activate ssh needs cancel-kill behavior. — source: reports/cloud-designer/40-lojix-cutover-plan/6-cutover-plan.md:1
- For production cutover, treat secrets parity as mandatory; secret-free zeus build/eval can be a non-authoritative Stack-B parity smoke but not a production substitute. — source: user@turn-prefix plus reports/cloud-designer/40-lojix-cutover-plan/2-secrets-gate-and-secret-free-nodes.md
- The OsOnly firmware divergence is a one-line build-correctness fix in lojix schema_runtime.rs around the firmware input generation; cloud research reported daemon_OsOnly true versus lojixCli_OsOnly false. — source: user@turn-prefix
- Spirit Observe is routine guidance; on current Lojix work, observe/refresh is appropriate but do not record the cloud-designer research as new intent. — source: user@turn-prefix
- Transmission Remote GTK is GTK3 and can require actual gtk-theme adw-gtk3-dark rather than only portal color-scheme prefer-dark; Chroma failures can leave dconf partially applied. — source: cmd:dconf read /org/gnome/desktop/interface/gtk-theme#before-adw-gtk3-after-adw-gtk3-dark
- Clean temporary VSCodium state plus disabled extensions can isolate profile/extension issues; absence of Claude/Codex in that clean state is expected, not itself a failure. — source: previous-brief:VSCodium clean-window result
- Operational corrections/status are not automatically Spirit intent; remove overcaptures and use conservative Spirit write-gate discipline. — source: previous-brief:Spirit discipline
- For Lojix, root workspace exposure needs a root Cargo.lock as well as Cargo.toml; otherwise root cargo commands ignore the triad-port lock and may cold-fetch missing private git dependencies. — source: cmd:cd /git/github.com/LiGoldragon/lojix && cargo test -p lojix --no-default-features#error-failed-to-get-nota-codec
- A source-level truth test can be false-positive if old API names remain in comments; make the test forbid old Actor* names and require Async* names so comments cannot satisfy the migration witness. — source: session experience: actor_native_runtime.rs initially passed because daemon comments still mentioned ActorMulti*

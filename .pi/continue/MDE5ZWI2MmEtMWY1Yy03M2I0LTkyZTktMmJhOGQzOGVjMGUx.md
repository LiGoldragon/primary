## Learned
- For DJI Mic dictation on Ouranos, binding Whisrs to a stable Pulse monitor name is insufficient for the CPAL/ALSA PipeWire path; resolving the hot monitor source to its live object.serial and exporting PIPEWIRE_NODE to that serial at daemon start works without polling. — source: session validation: arecord/Whisrs tests and CriomOS-home aa4e16c7
- Whisrs is not the root cause of DJI Bluetooth capture instability, but it is the production consumer and the right validation target because a hot PipeWire graph is not enough if STT still captures from the moving public Bluetooth source. — source: assistant answer to human clarification, anchored by live Whisrs source-output validation
- Broken user-level systemd slice symlinks can shadow valid /etc/systemd/user slice units; deleting only broken symlinks and daemon-reloading is non-disruptive, unlike forcing graphical session slice reconciliation. — source: cmd:rm ~/.config/systemd/user/session.slice background.slice; systemctl --user daemon-reload
- For Lojix installed owner-socket validation, running lojix-daemon as a separate system user would reject normal meta-lojix clients because OwnerPeerAuthority requires matching uid and gid; on Ouranos the service must run as li:users for the local operator client. — source: /git/github.com/LiGoldragon/lojix/tests/owner_peer_authority.rs and cmd:id li/getent group users
- Lojix daemon startup remains a binary/rkyv boundary even when human-facing clients need nota-text; packaging should preserve a daemon-only build pressure and prove inline NOTA startup rejection. — source: /git/github.com/LiGoldragon/lojix/flake.nix checks.daemon-startup-rejects-nota
- CriomOS modules should gate lojix-daemon installation on semantic Horizon service vocabulary such as PersonaDevelopment rather than hard-coding Ouranos or Zeús host names. — source: /git/github.com/LiGoldragon/CriomOS/modules/nixos/lojix.nix and /git/github.com/LiGoldragon/goldragon/datom.nota
- rust-overlay stable.latest is a moving fixed-output target and caused channel-rust-stable hash drift during FullOS evaluation; pin exact Rust versions in the profile toolchain rather than relying on latest. — source: /git/github.com/LiGoldragon/CriomOS-home/packages/rust-toolchain/default.nix commit ce31cc21
- Prometheus remote-builder FullOS Eval for Ouranos hit rust-overlay channel fixed-output hash drift even after several lock updates; local builder None was acceptable for installing on local Ouranos, while Zeús Build/Boot through the installed daemon with builder prometheus succeeded. — source: cmd:lojix-run FullOs ouranos builder prometheus failures, then builder None successes, then meta-lojix Zeus Build/Boot successes
- Installed lojix-daemon can return Deployed for Zeús Eval/Build/Boot and update the host boot profile while the ordinary query still shows an empty generation listing; treat live-set/query parity as a separate state-plane bug, not as hidden success. — source: cmd:meta-lojix Zeus Boot#Deployed plus cmd:lojix Query#(Queried ([] (6 6))) and bead primary-htrk
